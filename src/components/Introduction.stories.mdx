## Get Started

### Atomic Design

We follow the [**Atomic Design**](https://bradfrost.com/blog/post/atomic-web-design/) principles. Basic building blocks of the UI are assigned to **ðŸ’§ Atoms**, **ðŸŒ± Molecules** and **ðŸŒ³ Organisms** depending on their complexity and dependencies. ### Figma

Storybook is where we write the code on the basis of what designers have build in the represetnative Figma file [**anynines Design System**](https://www.figma.com/file/FquPS1rVsEsTOPxR8SCw04/anynines-Design-System?node-id=77%3A521). Figma is where we think, concept, design and prototype as a team, while storybook is the place where developers can check out all the UI components. It is important that both sides stay in synch whenever possible.

### Material UI

Since we only have a small team to provide a wide range of UI components we need to build on something **bullet proven**, **maintained** and **easy to extend**. Our choice for that one is **Material UI** (Mui) from Google. We simply take all the basic components from MUI wrap them to override styles, add default props or add additional props.

### Toolabs

To manage different themes with following the same structure we use a tool called toolabs. It is a graphical UI to manage colors, fonts and typo, icons, images and even ui texts. The data we configure there can be synched with Figma and can be exported as javascript files to be used in any app.
